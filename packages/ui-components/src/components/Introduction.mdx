# Welcome my portfolio Component Library!

Why use a component library you ask? Well simply put, it's a great way to to show how building in an organized manner, can help speed up the overall development experience! By following a [**component-driven**](https://componentdriven.org) process, using atomic components with no self contained state or data, we are able to manage much better components and services in our sites and applications!

In my library you will be able to see my skill in writing clean and concise React components, how I persue styling, naming systems, simple documentation, ans all while trying to follow a SOLID standard as well as TDD, all applied for business logic first!
